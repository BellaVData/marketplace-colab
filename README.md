# Анализ данных маркетплейса

## Цель проекта
Основной целью исследования является комплексный анализ операционных показателей маркетплейса для выявления ключевых трендов, оценки эффективности продаж и понимания поведения клиентов.

**Задачи проекта:**
1.  **Предобработка данных:** Очистка и подготовка датасета к анализу.
2.  **Временной анализ:** Установление периода исследования и изучение динамики продаж.
3.  **Анализ статусов:** Оценка распределения заказов по состояниям (завершен, отменен, возврат).
4.  **Расчет бизнес-метрик:** Вычисление среднего чека и общего объема продаж (GMV).
5.  **Ранжирование:** Формирование рейтинга наиболее востребованных товаров.
6.  **Факторный анализ:** Оценка влияния методов оплаты на уровень отмен и возвратов.

## Исходные данные
Для анализа использовался датасет `marketplace_dataframe.csv` объемом более 140 тысяч записей.
**Основные поля данных:**
* Информация о товаре (ID, артикул, категория, цена).
* Данные о заказе (статус, количество, итоговая сумма, скидка).
* Транзакционные данные (метод и дата оплаты).
* Данные о клиенте (ID клиента, дата начала обслуживания).

## Используемые методы и библиотеки
 **Библиотеки:** `pandas`, `numpy` для обработки данных; `matplotlib`, `seaborn` для визуализации (подразумевается задачами анализа).
 **EDA (Разведочный анализ):** Первичный обзор структуры, работа с пропусками, удаление служебных столбцов и нормализация названий.
 **Агрегация и группировка:** Использование функций `groupby`, `sum`, `mean` для расчета ключевых показателей по категориям и брендам.
 **Time Series Analysis:** Преобразование типов данных в `datetime` для анализа динамики по времени.
